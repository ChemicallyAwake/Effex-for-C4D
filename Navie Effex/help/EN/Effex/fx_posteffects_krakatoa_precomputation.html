Precomputations

In the following all description content that is displayed in italic is a quote taken from the thinkbox Krakatoa MY user reference manual which you can find here: http://www.thinkboxsoftware.com/krakatoa-my-render-settings/

It is generally a great resource for getting to know Krakatoa better.

Advection [sec]



If this value is 0.0 it is deactivated.
Any value above 0.0 will advect the particles using their velocity property.

The value is given in seconds.

Reduce before repopulation



If checked it activates the reduction algorithm of KrakatoaSR.

Reduce particles to..



Reduces the density by removing particles from the input particles.

Precomputations::Repopulate
Contains settings for the KrakatoaSR repopulation algorithm, which produces new particles around existing particles to increase the visual density.

It is not an intelligent hole filling algorithm so don't expect it to be.
And according to thinkbox it is best suited for liquid simulations with even particle distributions (which you pretty much get by using the weak spring operator in your simulations).

Seed



The seed value for the random placement of particles. This is like a starting index inside the random sequence array.

Fill radius



This is the space around particles (imagine a sphere) considered for the generation of new particles.

Particle emission subdivisions



The Particle emission subdivisions defines the Voxel Spacing to use as a fraction of the Fill Radius.
In other words, a value of 1 means that if the Fill Radius has a value of 1.0, this value will be divided once, producing a Spacing of 0.5 units.
Each subdivision increases the number of voxels in a cubic manner, in other words, Subdiv 1 means 2x2x2 = 8 voxels within the Fill Radius volume, a value of 2 means 3x3x3=27 voxels and so on.

So this can quickly result in an exponential increment of particle counts and takes longer to process.

Particles per subdivision



The Particles per subdivision value defines the number of particles to seed randomly inside each Voxel. Increasing the number two times (e.g. from 1 to 2 particles) will produce roughly twice as many particles (some particles might not be generated if inside a Voxel but outside of the Fill Radius).

Density falloff



The Density Falloff value can range from 0.0 to 1.0 and defines the falloff of the Density channel based on the distance from the "surface" of the volume defined by all Fill Radii.
When the value is 0.0, all particles will get the same Density regardless of their distance to the surface. A value of 1.0 produces a linear falloff. A value of 0.5 produces constant density if the new particle is farther away from the surface than half of the Fill Radius value, and linear falloff in the range from half the distance to the actual surface.
